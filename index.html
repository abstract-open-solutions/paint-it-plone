Ratio

The relationship in quantity, amount, or size between two or more things — Definition of ratio

Web pages have to adapt to different sizes and resolutions, making absolute measurement very difficult. But you can try to maintain a pattern with proportions, which is fundamental for:

  • Legibility
  • Usability
  • Aesthetics

With something as fluid as HTML, proportions are what matters, not fixed values.

Always reason in terms of proportions and how to vary them

What to measure

  1. Font sizes (headings and normal text)
  2. Paddings, margins, interlines
  3. Elements
  4. Lines (borders, graphic signs, etc)

Start measuring font sizes and implement them, and then try to map every other measure into proportions of the above.

Careful with line heights, margins, etc.

Typography

And much of what I stumbled into by following my curiosity and intuition turned out to be priceless later on. Let me give you one example: Reed College at that time offered perhaps the best calligraphy instruction in the country. Throughout the campus every poster, every label on every drawer, was beautifully hand calligraphed. Because I had dropped out and didn’t have to take the normal classes, I decided to take a calligraphy class to learn how to do this. I learned about serif and san serif typefaces, about varying the amount of space between different letter combinations, about what makes great typography great. It was beautiful, historical, artistically subtle in a way that science can’t capture, and I found it fascinating.

None of this had even a hope of any practical application in my life. But ten years later, when we were designing the first Macintosh computer, it all came back to me. And we designed it all into the Mac. It was the first computer with beautiful typography. If I had never dropped in on that single course in college, the Mac would have never had multiple typefaces or proportionally spaced fonts.

Typography is harder than you thought.

It was complex back in the print days, and got even more complex with computer and screens

LESS vs SASS

LESS

  • Easier to setup
  • Works client side (great for development)
  • Lacks a couple features
  • Javascript & node.js

LESS is easier to get started, is great for development due to client-side mode.

Javascript tends to be hard to read, compilers are hard, a compiler written in Javascript is...

SASS

  • Feature complete
  • Slightly more difficult to set up
  • Ruby

SASS has every functionality, but is harder to set up.

I chose LESS because I know Javascript but not Ruby.

No reference implementation in python.

Non-reference implementation often lack behind (small communities etc)

It provides a documented set of widgets, where it's easy to pick the one that fits best thanks to visual examples, and just using the correct markup does the job.

Diazo tricks

A few parts of Diazo often overlooked or misunderstood.

Wrapping stuff

<replace css:content="#my-content">
  <div id="my-wrapper" class="a-class">
    <div class="another-class">
      <xsl:apply-templates select="node()" />
    </div>
  </div>
</replace>

if ... elif ... else

<rules css:if-content="#foo"> ... </rules>
<rules css:if-content="#bar"> ... </rules>
<rules css:if-not-content="#foo, #bar"> ... </rules>

classA is classB

<replace css:content=".classA">
  <xsl:copy select="node()">
    <xsl:attribute name="class">
      <xsl:value-of select="./@class" /> classB
    </xsl:attribute>
    <xsl:apply-templates />
  </xsl:copy>
</replace>

and

<rules css:if-content="#foo">
  <rules css:if-content="#bar"> ... </rules>
</rules>
